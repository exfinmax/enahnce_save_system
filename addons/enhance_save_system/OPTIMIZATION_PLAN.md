# Enhanced Save System — 功能总结与优化待办清单

## ✅ 已有功能总结（现有能力）

### 核心架构
- **模块化存档框架**：所有存档逻辑通过 `ISaveModule` 模块化，`SaveWriter` 统一收集和写盘。
- **双轨存档**：
  - `global.json`：全局数据（设置、按键绑定、统计等）。
  - `slot_XX.json`：槽位数据（关卡进度、玩家状态、存档槽数据）。
- **JSON 存储**：人可读、易调试，存档内含 `_meta`（版本、时间戳、游戏版本、加密标记）。

### 扩展模块示例
- `SettingsModule`：保存/加载游戏设置（音量、语言、全屏等）。
- `KeybindingModule`：保存/加载按键绑定（支持键盘/鼠标/手柄事件序列化）。
- 其他模块（Player/Level/Stats 等）演示如何扩展和使用 `ISaveModule`。

### 进阶功能（已更新）
- **模块配置文件**：支持 `save_modules.cfg` 控制模块加载顺序、启用/禁用及优先级。
- **版本迁移机制**：提供 `MigrationManager`，结合 `ISaveModule.migrate_payload()`，可自动将旧版本存档升级到最新格式。
- **原子写入与备份**：支持 `atomic_write_enabled` + `backup_enabled`，写盘时可使用临时文件/备份机制提升可靠性。
- **加密模式**：支持多种加密方式（`encryption_mode`），包含默认 `aes_gcm`；`encryption_enabled` 可切换开启。
- **压缩支持**：支持 `compression_enabled` + `compression_mode`（gzip/deflate）降低存档体积。
- **分模块存储**：可启用 `split_modules_enabled`，将每个模块的存档分成独立文件（利于并行读写、差异化更新）。

### 数据序列化与可扩展性
- `ResourceSerializer` 支持：
  - 自定义 `SaveResource` 类型的序列化/反序列化。
  - `InputEvent`（按键、鼠标按键、手柄按键/轴）的序列化/反序列化。
  - 深度序列化：支持 Array/Dictionary 嵌套的 `SaveResource`。

### 实用功能
- **自动注册模块**：启动时扫描 `Modules/` 目录并自动注册符合 `ISaveModule` 的模块。
- **可手动注册模块**：通过 `SaveSystem.register_module()`。
- **自动存档**：支持间隔存档、指定槽位存档、支持截图存档（用于存档预览）。
- **存档导入/导出**：支持将槽位导出到外部 JSON、从外部 JSON 导入覆盖槽位。
- **状态查询 / UI 友好**：`list_slots()` 返回槽位元信息（存在性、meta、截图路径），以及各种信号事件（保存/加载完成等）。

## 🧰 优化待办清单（待改进项）

### 1) 安全性：加密机制增强（高优先）
- 当前支持 `aes_gcm`，以及可配置模式。
- 建议：
  - 增加密钥派生 / 随机盐支持（避免直接使用固定文本作为 key）。
  - 支持多种密钥来源（项目设置、用户输入、平台密钥管理）。
  - 在 `_meta` 记录加密模式与参数，便于排查兼容性。

### 2) 可靠性：写入原子性与备份（高优先）
- 已有 `atomic_write_enabled` + `backup_enabled` 配置，但需验证所有存档类型（全局/槽位/分模块）都正确生效。
- 建议：
  - 增加写入失败检测与自动回滚。
  - 支持“保留 N 份历史备份”以便调试/手动恢复。

### 3) 版本兼容：迁移策略与测试（中优先）
- 已有 `MigrationManager` 与 `ISaveModule.migrate_payload()` 钩子。
- 建议：
  - 提供示例迁移脚本（如 `migration_demo.gd`）并纳入项目流程。
  - 增强错误日志与回滚策略，防止迁移失败导致存档损坏。

### 4) 结构与性能：分模块存储与压缩（中优先）
- 已支持分模块文件存储（`split_modules_enabled`）与压缩（gzip/deflate）。
- 建议：
  - 提供可控策略：按模块/按阈值启用分模块与压缩。
  - 验证压缩 + 加密组合在读写性能上的影响。

### 5) 用户体验：存档管理工具（中/低优先）
- 目前只有脚本接口，缺少内置存档选择界面。
- 建议：
  - 在 Demo 场景中新增“存档管理” UI，显示 `list_slots()` 结果并提供读写/删除/导入/导出操作。
  - 提供编辑器插件面板（可查看/删除存档、查看 `_meta`、查看截图、选择载入）。

### 6) 模块注册灵活性（低优先）
- 已支持 `save_modules.cfg` 配置，可控制模块启用和加载优先级。
- 建议：
  - 支持对某个模块开启/禁用、按条件加载（平台/调试模式）。
  - 提供运行时 API 以动态启用/禁用模块（有助于热修复/调试）。
